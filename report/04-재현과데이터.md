## 3. 재현 패턴

```bash
# 1) 서빙 (AR): nightly, prefix-cache off, reasoning-parser
docker run -d --name gd --gpus all --ipc=host \
  -v ~/.cache/huggingface:/root/.cache/huggingface \
  -e VLLM_MEMORY_PROFILER_ESTIMATE_CUDAGRAPHS=0 -p 8000:8000 \
  vllm/vllm-openai:nightly --model google/gemma-4-31B-it \
  --max-model-len 40960 --gpu-memory-utilization 0.92 \
  --no-enable-prefix-caching --reasoning-parser gemma4 [--quantization fp8] [--kv-cache-dtype fp8]

# 2) thinking 프록시
ENABLE_THINK=true setsid python3 inject_proxy.py &   # :8001 → :8000

# 3) MTP: --speculative-config 추가 (Gemma=별도 드래프터, Qwen=임베디드 head)
#   '{"model":"google/gemma-4-31B-it-assistant","num_speculative_tokens":4}'
#   '{"method":"mtp","num_speculative_tokens":2}'

# 4) 평가 (전부 base_url=:8001 경유, max_gen_toks=32768, temp0.6/top_p0.95)
#   lm-eval GPQA(full) / inspect mmlu_pro·ifeval(--epochs 1) / run_hret.py haerae·kmmlu

# 5) 속도 / 컨텍스트
python3 bench_tp.py <label>          # short + 8K
python3 niah.py <label> 0 0.1,0.5,0.9
```

**재개 가능 설계:** 셀·벤치별 `$CAMP/.done_*` 마커로 skip, 셀별 즉시 CSV append, 캠페인은 `setsid`로 detached·파일로깅(Spot 선점/sshd wedge 대비). 폴링은 가벼운 sudo 읽기만.

**메모리 가이드(H100 80GB):** dense 31B bf16는 kv-fp8 필수(아니면 94K cap), MTP는 OOM 경계. fp8/qat는 util 0.90 여유. diffusion `--max-num-seqs 4`. qwen35 bf16/int8은 graph 불가 → eager(15tok/s, 비교 무효).

## 4. 서빙 설정 매트릭스 (모델별 정본)

> **`--reasoning-parser`는 모델 계열로 결정된다** — 가장 자주 헤매는 지점. Gemma 계열(`gemma-4-*`)=**`gemma4`**, Qwen 계열(`Qwen3.6-*`)=**`qwen3`**, **diffusiongemma=파서 없음**(미지정; thinking이 content에 섞여 나옴 → 마지막 답 기반 추출). 파서가 틀리면 thinking이 content에 안 발리거나 답까지 제거돼 점수가 깨진다.

| 모델 | model id (fp8 / base) | 이미지 | parser | 정밀도 플래그 | MTP 설정 | 특수 플래그 |
|---|---|---|---|---|---|---|
| Gemma 12B dense | `google/gemma-4-12B-it` | nightly | `gemma4` | bf16(없음) · fp8(`--quantization fp8`) · qat(`google/gemma-4-12B-it-qat-w4a16-ct`, 플래그 없음) | 드래프터 `google/gemma-4-12B-it-assistant`, nst 4 | — |
| Gemma 26B-A4B MoE | `google/gemma-4-26B-A4B-it` | nightly | `gemma4` | bf16 · fp8 · int8(`--quantization int8_per_channel_weight_only`, MoE 실효) | 드래프터 `google/gemma-4-26B-A4B-it-assistant`, nst 4 | — |
| Gemma 31B dense | `google/gemma-4-31B-it` | nightly | `gemma4` | bf16(**`--kv-cache-dtype fp8` 필수**) · fp8 · qat(`google/gemma-4-31B-it-qat-w4a16-ct`) | 드래프터 fp8=`...-31B-it-assistant`, qat=`google/gemma-4-31B-it-qat-q4_0-unquantized-assistant`, nst 4 | bf16+MTP=OOM(NA) |
| diffusiongemma 26B-A4B | `google/diffusiongemma-26B-A4B-it` | **gemma** | **없음** | bf16 · fp8 · int8 | 없음 | **`--max-num-seqs 4`**, max-model-len 16384~49152(canvas) |
| Qwen 27B dense-hybrid | `Qwen/Qwen3.6-27B-FP8` / `Qwen/Qwen3.6-27B` | **v0.22.1**(nightly 회귀) | `qwen3` | bf16 · fp8 · int8(속도이득 0) | 임베디드 head `{"method":"mtp","num_speculative_tokens":2}` | **`--max-num-seqs 256`**(gated-delta SSM 캐시; ≤783) |
| Qwen 35B-A3B MoE | `Qwen/Qwen3.6-35B-A3B-FP8` / `Qwen/Qwen3.6-35B-A3B` | nightly | `qwen3` | **fp8만 실용** · bf16/int8(70GB→eager 15tok/s) | 임베디드 head `{"method":"mtp","num_speculative_tokens":2}` | bf16/int8: `--gpu-memory-utilization 0.97 --enforce-eager`(+kv-fp8) |

**MTP 두 방식:** Gemma=**별도 드래프터 모델**(`--speculative-config '{"model":"<id>-assistant","num_speculative_tokens":4}'`), Qwen=**임베디드 MTP head**(`--speculative-config '{"method":"mtp","num_speculative_tokens":2}'`).

**공통 docker 인자:** `--gpus all --ipc=host -v ~/.cache/huggingface:/root/.cache/huggingface -e HF_TOKEN=$HF_TOKEN -e VLLM_MEMORY_PROFILER_ESTIMATE_CUDAGRAPHS=0 -p 8000:8000 --no-enable-prefix-caching`. 정확도 평가 시 `--reasoning-parser` 추가 + thinking 프록시(:8001) 경유. 속도 측정 시 파서·프록시 불필요.

**이미지 선택 규칙:** Gemma(AR/MTP)·Qwen35 = `nightly`. **Qwen27 = `v0.22.1`**(nightly에서 `Qwen3_5MTP` 회귀로 DIED). diffusion = `gemma`. 단일 런타임이 전 모델을 못 덮으므로 모델별로 고정.
